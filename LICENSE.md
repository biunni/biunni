## 🔧 Common Issues and Fixes

### ❌ "Failed to Attach" / Injection Error
**Cause:** Roblox updated after your current Bunni version was released, or antivirus is blocking the loader process.
**Fix:** Restart Bunni — the auto-updater will pull the latest patch automatically. If the issue persists, temporarily disable real-time antivirus protection and retry.

---

### ❌ Script Runs but Has No Effect In-Game
**Cause:** The script was written for an older Roblox API version, or it targets a game that has since patched the exploit it relies on.
**Fix:** Try a different script from the built-in Script Hub to confirm Bunni itself is working. If Hub scripts execute fine, the issue is with the specific script, not the executor.

---

### ❌ Bunni Won't Launch on Windows (Crash on Startup)
**Cause:** Missing .NET 6.0 runtime or outdated Visual C++ Redistributable.
**Fix:** Install the latest [.NET Runtime](https://dotnet.microsoft.com/download) and [Microsoft Visual C++ Redistributable](https://aka.ms/vs/17/release/vc_redist.x64.exe), then relaunch Bunni as Administrator.

---

### ❌ Android APK Fails to Install
**Cause:** "Unknown Sources" / "Install Unknown Apps" permission not granted, or insufficient storage.
**Fix:** Go to Settings → Security → enable "Install Unknown Apps" for your file manager. Ensure at least 200 MB of free internal storage is available.

---

### ❌ iOS Build Crashes Immediately on Launch
**Cause:** Developer profile not trusted, or the sideloaded IPA has expired (Apple enforces 7-day re-signing for free accounts).
**Fix:** Go to Settings → General → VPN & Device Management → find the Bunni developer profile → tap Trust. If it's been more than 7 days, re-sideload using Sideloadly or AltStore.

---

### ❌ Auto-Update Not Triggering
**Cause:** Outbound connections to Bunni update servers are being blocked by a firewall or corporate network.
**Fix:** Add `BunniLoader.exe` to your firewall's allow list. On restrictive networks, try using a VPN or switching to a mobile hotspot temporarily.
